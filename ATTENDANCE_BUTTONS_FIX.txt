================================================================================
Date: May 2, 2026
Status: FIXED ✓

================================================================================
ISSUE IDENTIFIED
================================================================================

Problem: Check In, Break, and Meeting buttons were not working

Root Cause: The frontend was not sending the correct parameters to the backend:
1. Using `user?.id` instead of `user?.userId` (backend expects userId)
2. Not sending `orgId` (backend requires it)
3. User object didn't have `userId` and `orgId` properties

================================================================================
SOLUTION IMPLEMENTED
================================================================================

1. UPDATED USER INTERFACE (AuthContext.tsx):
   - Added `userId` property to User interface
   - Added `orgId` property to User interface
   - These are now extracted from JWT token and response

2. UPDATED LOGIN FUNCTION (api.ts):
   - Extract `userId` from JWT token payload
   - Extract `orgId` from JWT token or response
   - Store both in user object
   - Pass to frontend components

3. UPDATED ATTENDANCE HANDLERS (Dashboard.tsx):
   - handleCheckIn: Now sends correct userId and orgId
   - handleCheckOut: Now sends correct userId and orgId
   - handleBreak: Now sends correct userId and orgId
   - handleMeeting: Now sends correct userId and orgId

================================================================================
BACKEND PARAMETERS REQUIRED
================================================================================

All attendance endpoints require:
- userId: User ID from JWT token
- employeeId: Employee ID from dashboard data
- orgId: Organization ID from JWT token
- location: Office/Remote (optional)
- notes: Description (optional)

Example Request Body:
{
  "userId": "user_id_from_jwt",
  "employeeId": "employee_id_from_db",
  "orgId": "org_id_from_jwt",
  "location": "Office",
  "notes": "Check-in from dashboard"
}

================================================================================
JWT TOKEN STRUCTURE
================================================================================

The JWT token contains:
{
  "userId": "user_id",
  "email": "user@example.com",
  "role": "employee",
  "orgId": "organization_id"
}

Frontend extracts these by:
1. Splitting token by '.'
2. Decoding the payload (second part)
3. Parsing as JSON
4. Extracting userId and orgId
