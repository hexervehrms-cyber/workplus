ROOT CAUSE IDENTIFIED
================================================================================

Problem: Buttons not working, 401 Unauthorized errors in backend logs

Root Cause: Token was not being stored after login!

The backend returns the token in:
  response.data.token

But the frontend was checking for:
  response.token

This mismatch meant the token was never stored in localStorage, so all
subsequent API requests were sent without authentication headers, resulting
in 401 Unauthorized errors.

================================================================================
BACKEND RESPONSE STRUCTURE
================================================================================

The backend login endpoint returns:

{
  "success": true,
  "message": "Login successful",
  "data": {
    "user": {
      "id": "user_id",
      "name": "User Name",
      "email": "user@example.com",
      "role": "employee",
      "avatar": "...",
      "organization": "...",
      "tenantId": "org_id"
    },
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": "24h"
  }
}

The token is in response.data.token, NOT response.token!

================================================================================
SOLUTION IMPLEMENTED
================================================================================

File Modified: frontend/src/app/utils/api.ts

Updated AuthService.login() to:
1. Check for token in response.data.token (correct location)
2. Fallback to response.token (for compatibility)
3. Extract user from response.data.user
4. Extract refreshToken from response.data.refreshToken
5. Store token in localStorage using TokenManager.set()
6. Store refreshToken in localStorage using TokenManager.setRefreshToken()
7. Extract userId and orgId from JWT token payload
8. Store complete user object with all properties

================================================================================

After login, localStorage should contain:

1. token: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
   - JWT token for authentication
   - Sent with every API request in Authorization header

2. refreshToken: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
   - Used to refresh token when it expires
   - Stored for session persistence

3. user: {
     "id": "user_id",
     "userId": "user_id_from_jwt",
     "name": "User Name",
     "email": "user@example.com",
     "role": "employee",
     "avatar": "...",
     "organization": "...",
     "tenantId": "org_id",
     "orgId": "org_id_from_jwt"
   }
   - User information
   - Used for API requests

================================================================================
API REQUEST FLOW
================================================================================

1. Frontend calls API endpoint
2. TokenManager.get() retrieves token from localStorage
3. Token is added to Authorization header: "Bearer {token}"
4. Request is sent to backend with token
5. Backend validates token
6. If valid: Request is processed
7. If invalid/missing: 401 Unauthorized error

Before fix: Token was not in localStorage → 401 errors
After fix: Token is stored → Requests work
