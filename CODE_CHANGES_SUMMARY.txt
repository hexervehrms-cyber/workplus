================================================================================
BACKEND ENDPOINTS USED
================================================================================

ENDPOINT 1: POST /api/employees
Purpose: Create employee user
Authentication: Required (JWT token)
Body Parameters:
  - name (string, required)
  - email (string, required)
  - password (string, required)
  - employeeCode (string, optional)
  - designation (string, optional)
  - department (string, optional)
  - baseSalary (number, optional)
  - joiningDate (date, optional)

Response:
  {
    success: true,
    data: {
      employee: { _id, name, email, role, ... }
    }
  }

ENDPOINT 2: POST /api/users
Purpose: Create admin/manager/hr/accountant user
Authentication: Required (JWT token)
Body Parameters:
  - name (string, required)
  - email (string, required)
  - password (string, required)
  - role (string, optional, default: 'employee')
  - organization (string, optional)
  - isActive (boolean, optional, default: true)

Response:
  {
    success: true,
    data: {
      _id: string,
      name: string,
      email: string,
      role: string,
      organization: string,
      isActive: boolean
    }
  }

ERROR HANDLING
================================================================================

TRY-CATCH BLOCK:

try {
  // Determine which endpoint to use
  if (formData.role === 'employee') {
    // Use employees endpoint
    response = await apiClient.post('/api/employees', {...});
  } else {
    // Use users endpoint
    response = await apiClient.post('/api/users', {...});
  }

  // Check if response is successful
  if (response.success) {
    // Extract user data
    const userData = response.data.employee || response.data;
    
    // Create new user object
    const newUser: GlobalUser = {...};
    
    // Add to list
    setUsers([...users, newUser]);
    
    // Close modal and reset form
    setShowAddForm(false);
    setFormData({...});
  }
} catch (err: any) {
  // Extract error message
  const errorMessage = err.response?.data?.message || 'Failed to create user';
  
  // Display error
  setError(errorMessage);
  
  // Log error
  console.error('Error creating user:', err);
} finally {
  // Stop loading
  setLoading(false);
}

  1. onChange handlers update formData state

WHEN USER CLICKS "CREATE USER":
  1. setLoading(true) - Show loading state
  2. setError(null) - Clear previous errors
  3. Validate form
  4. If invalid: setError(message) and return
  5. If valid: Call API

WHEN API RESPONSE RECEIVED:
  1. If success:
     - Create newUser object
     - setUsers([...users, newUser]) - Add to list
     - setShowAddForm(false) - Close modal
     - setFormData({...}) - Reset form
  2. If error:
     - setError(errorMessage) - Display error

FINALLY:
  1. setLoading(false) - Stop loading state
