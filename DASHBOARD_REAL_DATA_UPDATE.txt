================================================================================
WHAT'S NOW SHOWING REAL DATA
================================================================================

1. LEAVE BALANCE
   - Calculation: Total Annual Leave (20 days) - Used Leaves
   - Source: LeaveRequest collection (approved leaves this month)
   - Updates: Real-time when admin approves/rejects leave
   - Display: "X days" format

2. HOURS THIS WEEK
   - Calculation: Sum of hours worked from attendance records
   - Source: Attendance collection (today's hours worked)
   - Updates: Real-time when employee checks in/out
   - Display: "Xh" format

3. CURRENT SALARY
   - Calculation: Net salary from latest payslip
   - Source: Payroll collection (latest payslip)
   - Updates: When admin generates new payslip
   - Display: Formatted currency (USD, EUR, etc.)
   - Fallback: Base salary if net salary not available

4. PERFORMANCE SCORE
   - Calculation: Based on attendance percentage
   - Source: Attendance collection (present/absent status)
   - Updates: Real-time based on check-in status
   - Display: "X%" format

================================================================================
REAL-TIME UPDATES FLOW
================================================================================

1. ATTENDANCE CHECK-IN/CHECK-OUT:
   Employee checks in → Backend updates Attendance record
   → Socket.IO emits 'attendance:update' event
   → Frontend listener triggers fetchDashboardData()
   → KPI cards update with new hours worked

2. LEAVE APPROVAL:
   Admin approves leave → Backend updates LeaveRequest
   → Socket.IO emits 'leave:update' event
   → Frontend listener triggers fetchDashboardData()
   → Leave Balance KPI card updates

3. SALARY CALCULATION:
   Admin sets/updates salary → Backend updates Payroll record
   → Socket.IO emits 'dashboard:update' event
   → Frontend listener triggers fetchDashboardData()
   → Current Salary KPI card updates

4. PERFORMANCE UPDATES:
   Attendance status changes → Backend updates Attendance
   → Socket.IO emits 'attendance:update' event
   → Frontend listener triggers fetchDashboardData()
   → Performance Score KPI card updates

================================================================================
BACKEND DATA SOURCES
================================================================================

Endpoint: GET /api/dashboard/employee
File: backend/routes/dashboard-employee.js

Returns:
{
  employee: { id, name, designation, department, email, phone, avatar },
  attendance: {
    today: { status, checkIn, checkOut, hoursWorked },
    isPresent: boolean
  },
  leaves: {
    pending: number,
    approvedThisMonth: number,
    recent: [{ leaveType, status, startDate, createdAt, ... }]
  },
  expenses: {
    pending: number,
    totalThisMonth: number,
    recent: [{ category, amount, createdAt, ... }]
  },
  payroll: {
    latest: { id, month, year, baseSalary, netSalary, status }
  }
}

================================================================================
FRONTEND CALCULATIONS
================================================================================

Leave Balance:
- Total Annual Leave: 20 days (configurable)
- Used Leaves: data.leaves.approvedThisMonth
- Balance: Math.max(0, 20 - usedLeaves)
- Display: "${balance} days"

Hours This Week:
- Source: data.attendance.today.hoursWorked
- Display: "${hours}h"

Current Salary:
- Source: data.payroll.latest.netSalary (or baseSalary as fallback)
- Display: formatCurrency(salary)

Performance Score:
- Source: data.attendance.isPresent
- Calculation: isPresent ? 100 : 0
- Display: "${score}%"

================================================================================
REAL-TIME SOCKET.IO LISTENERS
================================================================================

The Dashboard component listens to three Socket.IO events:

1. 'attendance:update'
   - Triggered when: Employee checks in/out, takes break, enters meeting
   - Action: Calls fetchDashboardData() to refresh all metrics
   - Updates: Hours This Week, Performance Score

2. 'leave:update'
   - Triggered when: Admin approves/rejects leave request
   - Action: Calls fetchDashboardData() to refresh all metrics
   - Updates: Leave Balance, Upcoming Events

3. 'dashboard:update'
   - Triggered when: Any dashboard-related data changes
   - Action: Calls fetchDashboardData() to refresh all metrics
   - Updates: All KPI cards

REAL-TIME UPDATE FLOW DIAGRAM
================================================================================

Employee Action → Backend Update → Socket.IO Event → Frontend Listener
                                                    → fetchDashboardData()
                                                    → Update State
                                                    → Re-render KPI Cards

Example: Check-in Flow

2. Frontend sends POST /api/attendance/check-in

5. Frontend listener receives event
6. Frontend calls fetchDashboardData()
7. Frontend fetches latest dashboard data
8. Frontend updates kpiMetrics state
9. KPI cards re-render with new values
