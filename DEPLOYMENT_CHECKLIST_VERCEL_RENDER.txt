BACKEND DEPLOYMENT (RENDER)
URL: https://workplus-backend-sg3a.onrender.com

Pre-Deployment:
☐ backend/.env.production created with correct values
☐ render.yaml created
☐ backend/server.js verified
☐ All backend code committed to GitHub
☐ No sensitive data in code
☐ MongoDB connection string verified
☐ CORS_ORIGIN set to: https://workplus-murex.vercel.app

Deployment Steps (Option A - Dashboard):
☐ Go to https://dashboard.render.com
☐ Click "New +" → "Web Service"
☐ Connect GitHub repository
☐ Select repository
☐ Set Name to: workplus-backend
☐ Set Environment to: Node
☐ Set Build Command to: npm install
☐ Set Start Command to: node backend/server.js
☐ Select Plan: Standard (or higher)
☐ Add Environment Variables:
   ☐ NODE_ENV=production
   ☐ PORT=5000
   ☐ MONGODB_URI=(your MongoDB connection string)
   ☐ JWT_SECRET=(your JWT secret)
   ☐ CORS_ORIGIN=https://workplus-murex.vercel.app
   ☐ SMTP_HOST=smtp.office365.com
   ☐ SMTP_PORT=587
   ☐ SMTP_USER=hr@hexerve.com
   ☐ SMTP_PASS=(your SMTP password)
   ☐ FROM_EMAIL=hr@hexerve.com
   ☐ HR_EMAIL=hr@hexerve.com
☐ Click "Create Web Service"

Deployment Steps (Option B - render.yaml):
☐ Go to https://dashboard.render.com
☐ Click "New +" → "Infrastructure as Code"
☐ Select GitHub repository
☐ Render detects render.yaml
☐ Add Environment Variables in dashboard
☐ Deploy
