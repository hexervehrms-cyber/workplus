================================================================================
                              PROBLEM ANALYSIS
================================================================================

ERROR MESSAGE:
  ValidationError: Validation failed - title is required

ISSUE:
  When trying to create a new expense, you were getting a validation error
  because the 'title' field was set as required in the Expense model.

ROOT CAUSE:
  The title field was added to the Expense model with:
  title: { type: String, required: true }
  
  This meant:
  - All new expenses MUST have a title
  - Existing expenses without title would cause errors
  - Model validation was too strict

IMPACT:
  ❌ Could not create new expenses
  ❌ Got validation error on form submission
  ❌ Expenses table could not be populated

================================================================================
                              SOLUTION APPLIED
================================================================================

FILE MODIFIED:
  backend/models/Expense.js

CHANGE MADE:
  Changed title field from required to optional with default value

BEFORE:
  title: { type: String, required: true }

AFTER:
  title: { type: String, default: "" }

FIELD DETAILS:
  Field Name: title
  Type: String
  Required: No (optional)
  Default Value: Empty string ""
  Purpose: Stores the custom title for each expense

================================================================================
                              HOW IT WORKS NOW
================================================================================

VALIDATION FLOW:
  1. Frontend sends title field (if provided)
  2. Backend receives the request
  3. If title is provided → Stores the title
  4. If title is NOT provided → Uses default empty string ""
  5. Expense is created successfully

DISPLAY LOGIC:
  Frontend displays title with fallback:
  {expense.title || expense.description || 'Expense'}
  
  - If title exists → Shows title
  - If title is empty but description exists → Shows description
  - If both are empty → Shows "Expense"

BACKWARD COMPATIBILITY:
  - Old expenses without title → Will have empty title field
  - Old expenses will display description as fallback
  - Can be edited to add title later
