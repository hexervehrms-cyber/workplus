================================================================================
                    COMPLETE EMPLOYEE CREATION FLOW
================================================================================

1. Admin clicks "Add Employee" button
2. Modal opens with form
3. Admin fills employee details
4. Frontend validates all fields
5. API sends POST /api/employees request
6. Backend authenticates and authorizes
7. Backend validates input data
8. Backend hashes password
9. Backend creates User record
10. Backend creates Employee record
11. Backend emits Socket.IO events
12. Backend returns success response
13. Frontend receives response
14. Frontend shows success toast
15. Frontend closes modal
16. Frontend clears form
17. Socket.IO broadcasts employee_created event
18. All connected admins receive event
19. Employee list updates instantly
20. Dashboard count updates instantly
21. No page refresh needed

================================================================================
                         API ENDPOINTS
================================================================================

POST /api/employees
  - Create new employee
  - Requires: JWT token, admin/hr/super_admin role
  - Body: name, email, password, designation, department, baseSalary, phone

GET /api/employees
  - List all employees with pagination
  - Requires: JWT token
  - Query: page, limit, status, department, search

GET /api/employees/:id
  - Get single employee
  - Requires: JWT token

PUT /api/employees/:id
  - Update employee
  - Requires: JWT token, admin/hr/super_admin role

DELETE /api/employees/:id
  - Delete employee (soft delete)
  - Requires: JWT token, admin/hr/super_admin role

================================================================================
                      REAL-TIME SOCKET EVENTS
================================================================================

employee_created
  - Emitted when new employee is created
  - Broadcast to: tenant room, admin role room, hr role room
  - Frontend updates: employee list, employee count

employee_updated
  - Emitted when employee is updated
  - Broadcast to: tenant room, admin role room, hr role room
  - Frontend updates: employee list

employee_deleted
  - Emitted when employee is deleted
  - Broadcast to: tenant room, admin role room, hr role room
  - Frontend updates: employee list, employee count

dashboard_update
  - Emitted when dashboard metrics change
  - Broadcast to: tenant room
  - Frontend updates: KPI cards

================================================================================
                       DATABASE SCHEMA
================================================================================

User Collection:
  - _id (ObjectId)
  - name (String)
  - email (String, unique)
  - password (String, hashed)
  - role (String: employee, admin, hr, super_admin)
  - orgId (String)
  - isActive (Boolean)
  - createdAt (Date)
  - updatedAt (Date)

Employee Collection:
  - _id (ObjectId)
  - userId (ObjectId, ref: User)
  - employeeCode (String, unique)
  - designation (String)
  - department (String)
  - baseSalary (Number)
  - phone (String)
  - address (String)
  - status (String: active, inactive, terminated)
  - orgId (String)
  - createdAt (Date)
  - updatedAt (Date)
  - __v (Number, for optimistic locking)

================================================================================
                      SECURITY FEATURES
================================================================================

✅ Authentication
   - JWT tokens required for all requests
   - Token verification on every request
   - Token expiration (24 hours)
   - User existence check

✅ Authorization
   - Role-based access control (RBAC)
   - Only admin/hr/super_admin can create employees
   - Organization isolation
   - Employee can only see own data

✅ Password Security
   - Hashed with bcrypt (12 rounds)
   - Never stored in plain text
   - Never returned in API responses
   - Validated on login

✅ Data Validation
   - Required fields checked
   - Email format validated
   - Email uniqueness verified
   - Input sanitization

✅ Audit Logging
   - All actions logged
   - Includes: user, timestamp, action
   - Accessible for compliance
