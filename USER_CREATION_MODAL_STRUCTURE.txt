================================================================================
FIELD DESCRIPTIONS
================================================================================

FULL NAME (Required)
  - Type: Text input
  - Placeholder: "Enter user name..."
  - Validation: Non-empty string
  - Example: "John Doe"

EMAIL (Required)
  - Type: Email input
  - Placeholder: "user@example.com"
  - Validation: Valid email format, must be unique
  - Example: "john.doe@company.com"

PASSWORD (Required)
  - Type: Password input (masked)
  - Placeholder: "Enter password (min 6 characters)"
  - Validation: Minimum 6 characters
  - Example: "SecurePass123"
  - Note: Hashed on backend using bcrypt

ROLE (Required)
  - Type: Select dropdown
  - Default: "Employee"
  - Options:
    * Employee - Regular employee
    * Manager - Team manager
    * HR - Human Resources
    * Admin - Administrator
    * Accountant - Finance/Accounting
    * Super Admin - System administrator

ORGANIZATION (Optional)
  - Type: Text input
  - Placeholder: "Organization name..."
  - Default: "WorkPlus Inc."
  - Example: "Acme Corporation"

================================================================================
VALIDATION RULES
================================================================================

FULL NAME:
  ✓ Required
  ✓ Non-empty
  ✓ Any characters allowed
  ✗ Cannot be blank

EMAIL:
  ✓ Required
  ✓ Valid format (user@domain.com)
  ✓ Must be unique (no duplicates)
  ✗ Cannot be blank
  ✗ Invalid format
  ✗ Already exists

PASSWORD:
  ✓ Required
  ✓ Minimum 6 characters
  ✓ Any characters allowed
  ✗ Cannot be blank
  ✗ Less than 6 characters

ROLE:
  ✓ Required
  ✓ Must be from dropdown list
  ✗ Cannot be blank
  ✗ Invalid role

ORGANIZATION:
  ✓ Optional
  ✓ Any characters allowed
  ✓ Defaults to "WorkPlus Inc."

================================================================================
ERROR MESSAGES
================================================================================

ERROR 1: Missing Required Fields
  Message: "Name, email, and password are required"
  Trigger: User clicks "Create User" without filling required fields
  Action: Fill in all required fields and try again

ERROR 2: Password Too Short
  Message: "Password must be at least 6 characters"
  Trigger: User enters password with less than 6 characters
  Action: Enter password with 6 or more characters

ERROR 3: Duplicate Email
  Message: "User with this email already exists"
  Trigger: User enters email that's already in database
  Action: Use a different email address

ERROR 4: Invalid Email Format
  Message: "Invalid email format"
  Trigger: User enters invalid email format
  Action: Use valid email format (user@domain.com)

ERROR 5: Server Error
  Message: "Failed to create user"
  Trigger: Backend error during user creation
  Action: Check backend logs and try again

================================================================================
SUCCESS FLOW
================================================================================

1. User fills in all required fields
2. User clicks "Create User" button
3. Frontend validates form
4. Button shows "Creating..." state
5. API request sent to backend
6. Backend validates and creates user
7. User added to database
8. Response returned to frontend
9. New user added to list
10. Modal closes automatically
11. Form resets for next user
12. Success - User can now login

================================================================================
LOADING STATE
================================================================================

During user creation:
  - "Create User" button changes to "Creating..."
  - Button is disabled (cannot click again)
  - All form fields are disabled
  - Close button is disabled
  - User cannot interact with modal

After creation:
  - Modal closes automatically
  - Form resets
  - User appears in list
  - Ready for next user creation
